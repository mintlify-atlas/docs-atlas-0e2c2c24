## What is KeyForge?

KeyForge is a browser-based password generator built for people who take security seriously. Every password is generated entirely on your device using `crypto.getRandomValues()` — the same cryptographic source used by TLS and modern security systems. Nothing is sent to a server. Nothing is logged. Nothing is tracked.

### Four generation modes

<CardGroup cols={2}>
  <Card title="Password" icon="lock" href="/modes/password">
    Full control over length, character classes, anti-bruteforce guards, and custom character pools. Up to 128 characters with configurable minimums per class.
  </Card>
  <Card title="Passphrase" icon="text" href="/modes/passphrase">
    Word-based passwords drawn from a curated 200-word list. Memorable yet mathematically strong — with optional leet substitution, numbers, and symbols.
  </Card>
  <Card title="PIN" icon="grid-2" href="/modes/pin">
    Numeric PINs from 4–20 digits with guards against sequences, repeating digits, and common patterns like 1234 or 9876.
  </Card>
  <Card title="Phonetic Shield" icon="waveform" href="/modes/phonetic-shield">
    Pronounceable syllable-based passwords with entropy injected between groups. Say it aloud, remember it, and still defeat every bruteforce method.
  </Card>
</CardGroup>

## Why KeyForge?

<CardGroup cols={3}>
  <Card title="Cryptographic randomness" icon="dice">
    Uses `crypto.getRandomValues()` (CSPRNG) — not `Math.random()`. Every character selection is cryptographically unpredictable.
  </Card>
  <Card title="Anti-bruteforce guards" icon="shield-check">
    Automatically blocks keyboard walks (qwerty, asdf), dictionary fragments (pass, admin), sequential patterns, and ambiguous characters.
  </Card>
  <Card title="Zero telemetry" icon="eye-slash">
    No backend. No analytics. No cookies beyond a theme preference. Works offline once loaded.
  </Card>
  <Card title="Entropy DNA" icon="dna">
    Visual per-character breakdown of uppercase, lowercase, digit, and symbol composition with real-time entropy bits and crack-time estimates.
  </Card>
  <Card title="Phonetic Shield" icon="microphone">
    A unique mode — pronounceable passwords that resist dictionary, bruteforce, and pattern-based attacks simultaneously.
  </Card>
  <Card title="Full customization" icon="sliders">
    Configure symbol pools, excluded characters, forced inclusions, minimum counts per class, and more.
  </Card>
</CardGroup>

## Entropy at a glance

KeyForge displays real-time entropy for every generated credential:

| Strength Level | Entropy Range | Example Crack Time |
|---|---|---|
| Very Weak | < 36 bits | Milliseconds |
| Weak | 36–55 bits | Minutes to hours |
| Fair | 56–72 bits | Days to months |
| Strong | 73–94 bits | Years |
| Very Strong | 95–127 bits | Thousands of years |
| Fortress | 128+ bits | ∞ (heat death of universe) |

<Note>
Crack time estimates assume an offline attack at 1 trillion guesses per second (modern GPU farm, raw MD5/SHA). For bcrypt or Argon2, real-world times would be orders of magnitude longer.
</Note>
