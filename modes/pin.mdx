---
title: "PIN mode"
description: "Generate secure numeric PINs from 4–20 digits with guards against sequences, repeating digits, and common attack patterns."
---

PIN mode generates numeric-only codes for ATM cards, device unlock screens, 2FA backup codes, and any system that accepts only digits. All digits are drawn using `cryptoRand()`, ensuring uniform distribution with no bias.

<Warning>
  Short PINs have inherently low entropy regardless of how they are generated. A 4-digit PIN drawn from a pool of 10 digits has at most `4 × log₂(10) ≈ 13.3 bits` of entropy — secure only because most systems enforce rate-limiting or lockouts. Never use a short PIN to protect data in the absence of such controls.
</Warning>

## Length

<ParamField path="length" type="number" default="6">
  Number of digits in the PIN. Accepts values from **4 to 20**.
</ParamField>

### Entropy by length

| Digits | Max entropy | Notes |
|--------|-------------|-------|
| 4 | 13.3 bits | ATM minimum |
| 6 | 19.9 bits | Common device PIN |
| 8 | 26.6 bits | Strong device PIN |
| 10 | 33.2 bits | High-security numeric code |
| 12 | 39.9 bits | 2FA backup code range |
| 16 | 53.2 bits | Very strong |
| 20 | 66.4 bits | Maximum available |

## Security rules

<ParamField path="noRepeat" type="boolean" default="false">
  Require at least 7 unique digits in the PIN. For PINs shorter than 7 digits this rule requires every digit to be unique. For lengths ≥ 7 it requires `min(length, 7)` distinct digits, preventing outputs like `1111111111`.
</ParamField>

<ParamField path="noSequential" type="boolean" default="true">
  Reject PINs containing any run of three or more digits where each consecutive difference is 0 or ±1. This blocks:

  - Ascending runs: `123`, `234`, `345`, `456`, `567`, `678`, `789`
  - Descending runs: `987`, `876`, `765`, `654`, `543`, `432`, `321`
  - Repeated triplets: `111`, `222`, `333`, …, `000`

  The check slides a 3-digit window across the entire PIN, so a 6-digit PIN like `124567` is rejected because `456` and `567` appear within it.
</ParamField>

<ParamField path="noCommon" type="boolean" default="true">
  Reject PINs that contain any of the following known-weak 3-digit sequences anywhere in the output:

  `000` `111` `123` `234` `345` `456` `567` `678` `789` `987` `876` `765` `654` `543` `432` `321` `210`

  The check uses a sliding 3-digit window, so a 6-digit PIN is rejected if any three consecutive digits match a blocked sequence.
</ParamField>

## Fallback behavior

The generator retries up to **200 times** to produce a PIN that passes all active rules. If no valid PIN is found after 200 attempts — which can occur with very short lengths combined with strict rules — a fallback PIN is returned that was generated by pure random draw, skipping the strict checks. This ensures the UI always receives output.

<Note>
  The 200-attempt fallback is a safety net, not a common path. For typical configurations (6-digit PIN, default rules) a passing PIN is found on the first or second attempt.
</Note>

<Tip>
  For 2FA backup codes that users must write down, use 8–10 digits with `noRepeat` and `noSequential` enabled. The codes remain easy to transcribe while being resistant to guessing.
</Tip>
