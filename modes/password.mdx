---
title: "Password mode"
description: "Generate cryptographically secure passwords up to 128 characters with full control over character classes, anti-bruteforce guards, and custom character pools."
---

Password mode builds a random string from one or more character classes. Every character is drawn using the Web Crypto API (`crypto.getRandomValues`), so the output is cryptographically secure and suitable for any account, API key, or secret.

## Length

<ParamField path="length" type="number" default="16">
  Total number of characters in the generated password. Accepts values from **8 to 128**.
</ParamField>

<Tip>
  Aim for at least 16 characters for general-purpose accounts and 32+ for high-value secrets such as database credentials or encryption keys.
</Tip>

## Character classes

Enable or disable each class independently. At least one class must be active.

<ParamField path="useLower" type="boolean" default="true">
  Include lowercase letters **a–z** (26 characters).
</ParamField>

<ParamField path="useUpper" type="boolean" default="true">
  Include uppercase letters **A–Z** (26 characters).
</ParamField>

<ParamField path="useDigits" type="boolean" default="true">
  Include digits **0–9** (10 characters).
</ParamField>

<ParamField path="useSymbols" type="boolean" default="true">
  Include symbols from the active symbol pool (default: `!@#$%^&*()_+-=[]{}|;:,.<>?/~`, 30 characters).
</ParamField>

## Anti-bruteforce guards

These four checkboxes reject passwords that contain predictable patterns. If a candidate fails a check, generation retries (up to 200 attempts before a fallback).

<ParamField path="excludeAmbiguous" type="boolean" default="false">
  Remove visually similar characters **0, O, 1, l, I** from the pool to prevent transcription errors when reading passwords aloud or copying from a screen.
</ParamField>

<ParamField path="noRepeating" type="boolean" default="false">
  Reject passwords that contain any two identical characters in a row (e.g., `aa`, `11`).
</ParamField>

<ParamField path="noSequential" type="boolean" default="true">
  Reject passwords that contain keyboard walks (e.g., `qwerty`, `asdf`) or sequential character runs (e.g., `abc`, `xyz`, `123`, `321`). Implemented via `hasKeyboardWalk()` and `hasSequentialChars()`.
</ParamField>

<ParamField path="noWeakFragment" type="boolean" default="true">
  Reject passwords containing common weak fragments detected by `hasWeakFragment()`, such as dictionary words, common keyboard patterns, and repeated blocks.
</ParamField>

<Tip>
  All four guards are independent. Enabling more guards increases the average number of generation attempts but never weakens the final output — a password is only returned once it passes every active check.
</Tip>

## Minimum character counts

Guarantee that at least *n* characters of each class appear in the output. These settings are grouped in a collapsible panel in the UI.

<ParamField path="minLower" type="number" default="1">
  Minimum number of lowercase letters. Active only when `useLower` is enabled.
</ParamField>

<ParamField path="minUpper" type="number" default="1">
  Minimum number of uppercase letters. Active only when `useUpper` is enabled.
</ParamField>

<ParamField path="minDigits" type="number" default="1">
  Minimum number of digits. Active only when `useDigits` is enabled.
</ParamField>

<ParamField path="minSymbols" type="number" default="1">
  Minimum number of symbols. Active only when `useSymbols` is enabled.
</ParamField>

<Note>
  The sum of all minimums must not exceed the password length. If you set `minLower=4`, `minUpper=4`, `minDigits=4`, `minSymbols=4`, the password must be at least 16 characters long.
</Note>

## Custom character sets

These controls are grouped in a second collapsible panel in the UI.

<ParamField path="symbolSet" type="string" default="!@#$%^&*()_+-=[]{}|;:,.<>?/~">
  Replace the default symbol pool with any string of characters. Only takes effect when `useSymbols` is enabled.
</ParamField>

<ParamField path="customExclude" type="string" default="">
  Characters to remove from the final pool before generation. Useful for sites that ban specific symbols.
</ParamField>

<ParamField path="customInclude" type="string" default="">
  Characters that must appear in the password at least once, in addition to the active character classes. Each character in this string is appended to the mandatory set before random-fill.
</ParamField>

<Tip>
  Use `customExclude` to strip characters like `'`, `"`, and `` ` `` when a password will be embedded in a shell script or configuration file.
</Tip>

## How generation works

<Steps>
  <Step title="Build the pool">
    KeyForge assembles one pool per active character class (`lower`, `upper`, `digits`, `symbols`), applying `excludeAmbiguous`, `customExclude`, and the active `symbolSet`. A combined `all` pool is also built for random-fill.
  </Step>
  <Step title="Guarantee minimums">
    For each active class, `minX` characters are drawn with `cryptoRand()` and added to a `mandatory[]` array. Then every character in `customInclude` is appended (if it exists in the pool and space remains).
  </Step>
  <Step title="Fill remaining positions">
    Positions not yet filled by mandatory characters are populated by drawing randomly from the combined `all` pool until the array reaches the target `length`.
  </Step>
  <Step title="Fisher-Yates shuffle">
    The `mandatory[]` array is shuffled in-place using a CSPRNG-backed Fisher-Yates algorithm (`fisherYates()`), so mandatory characters appear at unpredictable positions.
  </Step>
  <Step title="Check anti-pattern guards">
    The candidate password is tested against every active guard (`noRepeating`, `noSequential`, `noWeakFragment`). If any check fails, the attempt is discarded and the process restarts from step 2.
  </Step>
  <Step title="Return or fall back">
    If a valid password is found within 200 attempts it is returned immediately. If all 200 attempts fail (only possible with very restrictive guard combinations on short lengths), a fallback password is generated by pure random draw — skipping strict checks — to ensure the UI always receives output.
  </Step>
</Steps>

## Entropy DNA bar and strength meter

Below the generated password, KeyForge displays:

- **Entropy DNA bar** — a color-coded visualization of the character-class distribution across each position. Each segment color maps to a class: lowercase, uppercase, digit, or symbol.
- **Strength meter** — calculated as `length × log₂(poolSize)` bits of entropy, where `poolSize` is the size of the combined active pool after exclusions. The meter labels correspond to:

| Bits | Label |
|------|-------|
| < 40 | Weak |
| 40–59 | Fair |
| 60–79 | Good |
| 80–99 | Strong |
| ≥ 100 | Very strong |

<Tip>
  A 16-character password with all four classes active has a pool of ≈ 92 characters, yielding `16 × log₂(92) ≈ 104 bits` — comfortably in the "Very strong" range.
</Tip>
