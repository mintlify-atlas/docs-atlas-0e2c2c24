KeyForge has no configuration file. All settings are exposed as UI controls on the page. The only value that persists between sessions is the theme preference, stored in `localStorage`.

<Note>
  Settings are not saved between sessions. Every time you open KeyForge, all generator options reset to their defaults. Only your theme preference (dark or light) is remembered.
</Note>

---

## Theme persistence

KeyForge stores exactly one value in `localStorage`:

| Key | Values | Default |
|-----|--------|---------|
| `kf-theme` | `'dark'` \| `'light'` | Follows `prefers-color-scheme` media query; falls back to `'dark'` |

```javascript
// Read the current theme
localStorage.getItem('kf-theme'); // 'dark' | 'light' | null

// Override programmatically
localStorage.setItem('kf-theme', 'light');
```

When no value is stored, KeyForge checks `window.matchMedia('(prefers-color-scheme: dark)')` and defaults to dark mode if the query is unsupported or returns `false`.

## Keyboard shortcut

<Tip>
  Press `Ctrl+Enter` (or `Cmd+Enter` on Mac) to regenerate a new password, passphrase, PIN, or phonetic password for whichever generator tab is currently active — without reaching for the mouse.
</Tip>

## No configuration file

KeyForge is a static single-page application with no server, no user accounts, and no persistent configuration storage beyond `localStorage`. There is no config file to edit and no environment variables to set.

If you are self-hosting KeyForge or embedding it, you can change the default behavior by modifying `pasgen.js` and `entropy.js` directly — all defaults are defined as JavaScript constants and function default parameters within those files.
