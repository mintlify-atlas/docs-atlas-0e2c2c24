KeyForge is designed so that your passwords never leave your device. The full policy is published at [keyforgegd.vercel.app/Privacypolit.html](https://keyforgegd.vercel.app/Privacypolit.html). This page summarizes its key commitments.

**Last updated:** 2026

## No data collection

KeyForge collects no data of any kind. All password and passphrase generation runs entirely on your device using the browser's built-in `crypto.getRandomValues()` API — the same cryptographic source used by TLS. No generated password, configuration choice, or usage pattern is ever transmitted anywhere.

There are no user accounts, no sign-up forms, and no input fields whose contents leave your browser.

---

## No cookies

KeyForge sets no cookies — tracking, analytics, session, or otherwise.

The only browser storage used is a single `localStorage` entry:

| Key | Purpose |
|-----|---------|
| `kf-theme` | Remembers your dark/light mode preference across visits |

This value is never read by any server. It exists purely to restore your theme preference when you reopen the app. You can clear it at any time via your browser's developer tools or by clearing site data.

---

## No telemetry

KeyForge contains zero telemetry, analytics, error reporting, or monitoring code. There are no calls to third-party tracking services. No events are fired to any external endpoint.

Your usage patterns — which generator modes you use, how many passwords you generate, what settings you choose — are never observed or recorded.

---

## Fully client-side architecture

KeyForge has no backend server. The application is a static bundle of HTML, CSS, and JavaScript files served from a CDN.

<Check>
  Passwords are generated entirely in your browser. They are never sent to any server, never stored in any database, and never logged anywhere.
</Check>

Because the app is entirely static, it also works offline once the page has loaded. You can save it locally and run it without an internet connection — generation will continue to work with full cryptographic security.

## Cryptographic security

All random values in KeyForge are generated by `crypto.getRandomValues()`, the browser's CSPRNG (Cryptographically Secure Pseudo-Random Number Generator). This is the same source of entropy used by TLS for key generation.

`crypto.getRandomValues()` is:

- **Cryptographically unpredictable** — output cannot be predicted from previous values
- **Seeded by the OS** — draws entropy from the operating system's entropy pool
- **Standardized** — specified by the W3C Web Cryptography API

KeyForge does not use `Math.random()` anywhere in its generation pipeline.
