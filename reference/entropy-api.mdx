---
title: "Entropy API"
description: "Complete reference for the window.Entropy object — KeyForge's entropy calculation and visualization engine."
---

`window.Entropy` provides entropy calculation, strength classification, crack-time estimation, and DOM visualization helpers. All calculation functions are pure and have no side effects.

```javascript
const { calcPasswordEntropy, getStrengthLevel, getCrackTime } = window.Entropy;
```

---

## Entropy calculations

### `getPoolSize(options)`

Calculates the effective character pool size after applying exclusions. This is the value `N` used in the entropy formula `H = L × log₂(N)`.

```javascript
const size = Entropy.getPoolSize({
  useLower: true,
  useUpper: true,
  useDigits: true,
  useSymbols: false,
  excludeAmbiguous: true,
});
// Returns effective pool size, e.g. 76
```

<ParamField path="options.useLower" type="boolean">
  Include lowercase letters in the pool count.
</ParamField>

<ParamField path="options.useUpper" type="boolean">
  Include uppercase letters in the pool count.
</ParamField>

<ParamField path="options.useDigits" type="boolean">
  Include digits in the pool count.
</ParamField>

<ParamField path="options.useSymbols" type="boolean">
  Include symbols in the pool count.
</ParamField>

<ParamField path="options.symbolSet" type="string">
  Custom symbol string. Defaults to `CHARSETS.symbols`.
</ParamField>

<ParamField path="options.excludeAmbiguous" type="boolean">
  Subtract ambiguous characters from the pool count.
</ParamField>

<ParamField path="options.customExclude" type="string">
  Additional characters to subtract from the pool count.
</ParamField>

**Returns**

<ResponseField name="poolSize" type="number">
  The effective pool size. Always at least `2` to prevent log₂(0) or log₂(1) edge cases.
</ResponseField>

---

### `calcPasswordEntropy(length, poolSize)`

Calculates entropy for a randomly generated password.

**Formula:** `H = length × log₂(poolSize)`

```javascript
const bits = Entropy.calcPasswordEntropy(16, 94);
// 16 × log₂(94) ≈ 104.9 bits
```

<ParamField path="length" type="number" required>
  Password length in characters.
</ParamField>

<ParamField path="poolSize" type="number" required>
  Effective character pool size (use `getPoolSize()` to compute this).
</ParamField>

<ResponseField name="entropy" type="number">
  Entropy in bits.
</ResponseField>

---

### `calcPassphraseEntropy(wordCount, wordListSize, hasNum, hasSym)`

Calculates entropy for a randomly generated passphrase.

**Formula:**
```
H = wordCount × log₂(wordListSize)
    + (hasNum ? log₂(90) : 0)
    + (hasSym ? log₂(7)  : 0)
```

```javascript
const bits = Entropy.calcPassphraseEntropy(5, 200, true, true);
// 5 × log₂(200) + log₂(90) + log₂(7) ≈ 50.8 bits
```

<ParamField path="wordCount" type="number" required>
  Number of words in the passphrase.
</ParamField>

<ParamField path="wordListSize" type="number" default="200">
  Size of the wordlist being sampled from. KeyForge's built-in wordlist contains 200 words.
</ParamField>

<ParamField path="hasNum" type="boolean" default="true">
  Whether a random digit is appended. Adds log₂(90) ≈ 6.5 bits.
</ParamField>

<ParamField path="hasSym" type="boolean" default="true">
  Whether a random symbol is appended. Adds log₂(7) ≈ 2.8 bits.
</ParamField>

<ResponseField name="entropy" type="number">
  Entropy in bits.
</ResponseField>

---

### `calcPinEntropy(length)`

Calculates entropy for a randomly generated numeric PIN.

**Formula:** `H = length × log₂(10)`

```javascript
const bits = Entropy.calcPinEntropy(6);
// 6 × log₂(10) ≈ 19.9 bits
```

<ParamField path="length" type="number" required>
  Number of digits in the PIN.
</ParamField>

<ResponseField name="entropy" type="number">
  Entropy in bits.
</ResponseField>

### `calcPhoneticEntropy(syllableCount, entropyLevel)`

Calculates entropy for a phonetic password generated by `PassGen.generatePhonetic()`.

The syllable space is derived from the consonant-vowel-consonant combinations available:

```
sylSpace = 39 × 10 × 17 + 39 × 10 = 7020

H = syllableCount × log₂(sylSpace)
    + injections × injectedChars × log₂(19)

Where injections = syllableCount (one injection between/after each syllable)
And injectedChars depends on entropyLevel:
  'light'  → 1
  'medium' → 2
  'heavy'  → 3
```

```javascript
const bits = Entropy.calcPhoneticEntropy(3, 'medium');
// 3 × log₂(7020) + 3 × 2 × log₂(19) ≈ 62.8 bits
```

<ParamField path="syllableCount" type="number" required>
  Number of syllable groups.
</ParamField>

<ParamField path="entropyLevel" type="string" required>
  Injection density. One of `'light'`, `'medium'`, or `'heavy'`.
</ParamField>

<ResponseField name="entropy" type="number">
  Entropy in bits.
</ResponseField>

## Strength classification

### `getStrengthLevel(entropyBits)`

Maps an entropy value to a named strength level with display metadata.

```javascript
const level = Entropy.getStrengthLevel(80);
// {
//   min: 73, max: 94,
//   label: 'Strong',
//   css: 'strong',
//   pct: 72
// }
```

<ParamField path="entropyBits" type="number" required>
  Entropy in bits.
</ParamField>

**Returns**

<ResponseField name="level" type="object">
  <Expandable title="properties" defaultOpen={true}>
    <ResponseField name="min" type="number">
      Lower bound (inclusive) of this level's entropy range.
    </ResponseField>
    <ResponseField name="max" type="number">
      Upper bound (inclusive) of this level's entropy range. `Infinity` for Fortress.
    </ResponseField>
    <ResponseField name="label" type="string">
      Human-readable strength label.
    </ResponseField>
    <ResponseField name="css" type="string">
      CSS class name applied to the strength bar and label.
    </ResponseField>
    <ResponseField name="pct" type="number">
      Percentage fill for the strength bar UI (0–100).
    </ResponseField>
  </Expandable>
</ResponseField>

**Strength levels**

| Label | Entropy range | Bar fill |
|-------|--------------|----------|
| Very Weak | 0–35 bits | 10% |
| Weak | 36–55 bits | 28% |
| Fair | 56–72 bits | 50% |
| Strong | 73–94 bits | 72% |
| Very Strong | 95–127 bits | 88% |
| Fortress | 128+ bits | 100% |

<Note>
  Entropy thresholds assume an offline attack at 1 × 10¹² guesses per second. A password rated "Strong" (73+ bits) would take centuries to crack under these conditions. See `getCrackTime()` for the full estimate.
</Note>

---

### `getCrackTime(entropyBits)`

Estimates how long a brute-force offline attack would take to crack a password with the given entropy.

**Assumption:** 1 × 10¹² guesses per second (high-end GPU array), average-case (2^(n-1) guesses needed).

```javascript
const time = Entropy.getCrackTime(80);
// e.g. "36 years"

Entropy.getCrackTime(40);  // e.g. "18 minutes"
Entropy.getCrackTime(128); // e.g. "5.4 × 10¹⁵ years"
```

<ParamField path="entropyBits" type="number" required>
  Entropy in bits.
</ParamField>

<ResponseField name="time" type="string">
  A human-readable time string such as `"3 seconds"`, `"14 hours"`, `"200 years"`, or `"5.4 × 10¹⁵ years"`.
</ResponseField>

## DOM helpers

These functions update existing DOM elements. They are used by KeyForge's UI and require specific element IDs to be present in the page.

### `updateStrengthUI(entropy, prefix)`

Updates the four strength indicator elements for a given generator mode. Element IDs are expected to follow the pattern `{elementType}-{prefix}`.

```javascript
// Updates elements:
// #strength-label-pwd
// #entropy-bits-pwd
// #crack-time-pwd
// #strength-fill-pwd
Entropy.updateStrengthUI(104.9, 'pwd');
```

<ParamField path="entropy" type="number" required>
  Entropy in bits to display.
</ParamField>

<ParamField path="prefix" type="string" required>
  ID suffix used to locate the target DOM elements. The following elements must exist:

  | Element ID | Updated content |
  |---|---|
  | `strength-label-{prefix}` | Strength label text (e.g. `"Strong"`) |
  | `entropy-bits-{prefix}` | Entropy value (e.g. `"104.9 bits"`) |
  | `crack-time-{prefix}` | Crack time estimate (e.g. `"36 years"`) |
  | `strength-fill-{prefix}` | `width` style of the strength bar element |
</ParamField>

<ResponseField name="undefined" type="void">
  No return value. Updates DOM in place.
</ResponseField>

---

### `renderEntropyDNA(password, containerId)`

Renders a color-coded cell-per-character visualization (the "DNA strip") into a container element. Each character gets a colored cell based on its class.

```javascript
Entropy.renderEntropyDNA('rK7#mP2@', 'dna-container-pwd');
```

| Character class | CSS class applied |
|---|---|
| Uppercase | `dna-upper` |
| Lowercase | `dna-lower` |
| Digit | `dna-digit` |
| Symbol | `dna-symbol` |

<ParamField path="password" type="string" required>
  The password string to visualize.
</ParamField>

<ParamField path="containerId" type="string" required>
  The `id` attribute of the DOM element to render into. Existing content is replaced.
</ParamField>

<ResponseField name="undefined" type="void">
  No return value. Renders cells into the container element.
</ResponseField>

---

### `renderDNALegend(containerId)`

Renders the color key legend for the DNA visualization — four labeled swatches for uppercase, lowercase, digit, and symbol.

```javascript
Entropy.renderDNALegend('dna-legend-pwd');
```

<ParamField path="containerId" type="string" required>
  The `id` attribute of the DOM element to render the legend into.
</ParamField>

<ResponseField name="undefined" type="void">
  No return value. Renders the legend into the container element.
</ResponseField>
